# 关于《三国演义》人物关系网络数据集的说明

本数据集用于项目：**基于图结构与 PageRank 算法的小说人物关系网络分析**。

## 1. 数据集性质

这是一个**人工整理型人物关系网络数据集**，适合用于：
- 图结构建模
- 邻接表存储
- 度数统计
- PageRank 人物重要性计算
- 人物关系网络可视化
- 研究报告和 PPT 展示

注意：本数据集不是严格的历史文献数据库，也不是自动文本挖掘结果，而是根据《三国演义》中主要人物关系、阵营关系、重要情节和叙事影响力整理而成。它的主要目标是为算法实践提供数据。

## 2. 文件说明

| 文件名 | 说明 |
|---|---|
| character_nodes.csv | 人物节点表，包含人物姓名、阵营、角色定位和简介 |
| character_edges_detailed.csv | 详细关系边表，包含人物 A、人物 B、权重、关系类型和说明 |
| character_edges_simple.csv | 简化关系边表，只包含 source、target、weight，适合 C++ 程序直接读取 |
| README_dataset.md | 当前说明文件 |

## 3. 数据规模

- 人物节点数：30
- 关系边数：102
- 图类型建议：加权无向图
- PageRank 处理建议：读取每条边后，在邻接表中同时加入 `source -> target` 和 `target -> source`

## 4. 权重含义

权重采用 1 到 5 的整数，表示两个人物关系强度或叙事关联强度：

| 权重 | 含义 |
|---|---|
| 5 | 核心关系，例如结义、父子继承、核心君臣、长期主要对手 |
| 4 | 重要关系，例如重要同盟、重要主从、关键战役对手 |
| 3 | 一般重要关系，例如同阵营同僚、阶段性对抗、重要外交关系 |
| 2 | 较弱关系，例如间接关联、时代传承、同阵营但直接互动较少 |
| 1 | 极弱关系，本数据集基本未使用，避免噪声过多 |

## 5. 建议的算法实验

可以在研究报告中设置以下实验任务：

1. 构建人物关系图；
2. 输出节点数和边数；
3. 统计每个人物的度数；
4. 计算加权度数；
5. 使用 PageRank 算法计算人物重要性；
6. 比较“度数排名”和“PageRank 排名”的差异；
7. 选取排名靠前的人物进行文学和网络结构解释。

## 6. 报告中可使用的建模表述

在本项目中，将《三国演义》中的人物抽象为图中的顶点，将人物之间的结义、君臣、同僚、亲属、联盟、敌对、战役冲突等关系抽象为图中的边。边的权重用于描述人物关系强度。由此形成一个加权人物关系网络，并使用 PageRank 算法分析人物在网络中的结构重要性。

## 7. C++ 读取建议

如果程序只需要读取关系数据，建议使用 `character_edges_simple.csv`：

```csv
source,target,weight
刘备,关羽,5
刘备,张飞,5
```

如果需要在报告中解释关系来源，建议同时参考 `character_edges_detailed.csv`。
